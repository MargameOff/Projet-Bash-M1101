Documentation Projet BASH V1.0

Les Choix:

-Les Boucles

Pour le projet, nous avons essentiellement concentré notre programme sur des boucles whiles en condition read line, permettant de rendre un code plus adaptatif au fichier Filtre.txt et au nombre de fichier que l’on doit analyser
On distingue cependant 2 types de boucles while en condition read line:

  • Les boucles dites “classique” : while read line
  • Les boucles avec suppression : while read line || [[ -n $line ]];

Les classiques sont celles vu en cours qui permettent de garder les variables utilisé
Cependant les boucles à suppression, elles, permettent de vider les variables utilisé dans celle-ci et également de la commencer à partir d’un pipe, elles agissent comme un sous-programme.

-Zenity

Nous avons également choisi d’utiliser les commandes de Zenity (inclus dans toutes les distributions Linux assez recente) permettant de générer des interfaces graphiques utilisateurs, permettant de simplifier notamment l’entrée des répertoires d’acces mais egalement pour des selections multilanguages ou des questions !

-Les Sous-Fonctions Pour l’Interface Terminal

Nous avons trouvé judicieux de partir sur cette solution pour la conceptions de l’interface, pour permettre un meilleur maintient de code mais également un système de répétition plus simple en cas de mauvais input.
